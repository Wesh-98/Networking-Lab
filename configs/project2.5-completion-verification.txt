## 🏗️ **Network Architecture**

### **Design:**
```
            Router (DHCP Relay Agent)
           /        |          \
    VLAN 10      VLAN 20      VLAN 99
    (Users)     (Servers)   (Management)
       |            |            |
      PC1          PC2      DHCP Server
                            (Referee)
```

### **DHCP Flow:**
```
1. PC1 broadcasts: "DHCP Discover"
2. Router receives on Gi0/0.10
3. Router forwards (unicast) to 192.168.99.10
4. Server sends "DHCP Offer"
5. Router forwards back to PC1
6. PC1 gets IP: 192.168.10.11 ✅
```

## 🔧 **Router Configuration**

### **DHCP Service Disabled:**
```cisco
Router(config)# no service dhcp
Router(config)# no ip dhcp pool VLAN10
Router(config)# no ip dhcp pool VLAN20
Router(config)# no ip dhcp pool VLAN99
```

### **Helper Addresses Configured:**
```cisco
! VLAN 10 Subinterface
interface GigabitEthernet0/0.10
 encapsulation dot1Q 10
 ip address 192.168.10.1 255.255.255.0
 ip helper-address 192.168.99.10

! VLAN 20 Subinterface
interface GigabitEthernet0/0.20
 encapsulation dot1Q 20
 ip address 192.168.20.1 255.255.255.0
 ip helper-address 192.168.99.10

! VLAN 99 Subinterface
interface GigabitEthernet0/0.99
 encapsulation dot1Q 99
 ip address 192.168.99.1 255.255.255.0
 ip helper-address 192.168.99.10
```

## 🎓 **Concepts Learned**

### **1. DHCP Relay Agent (ip helper-address):**
- Converts broadcast DHCP to unicast
- Forwards to specific server IP
- Essential for centralized DHCP
- Also relays: TFTP, DNS, TACACS, NetBIOS

### **2. Centralized DHCP Architecture:**
- One server handles all VLANs
- Easier to manage and monitor
- Reduces router workload
- Enterprise best practice

### **3. IP Address Management:**
- Reserved ranges for static devices
- DHCP pools start after reserved range
- Clear separation of duties
- Professional IP planning

### **4. Troubleshooting Skills:**
- Understanding old vs new IPs
- Verifying DHCP relay
- Testing connectivity methodically
- Reading DHCP server logs

## 🐛 **Issues Encountered & Solutions**

### **Issue 1: Command Syntax Error**
**Problem:** `show running-config interface gi0/0.10` gave error  
**Solution:** Use `show run int gi0/0.10` or `show run | section interface Gi0/0.10`  
**Lesson:** Learned correct Cisco IOS command syntax

### **Issue 2: Ping Failures to Old IPs**
**Problem:** Couldn't ping 192.168.10.2 or 192.168.20.2  
**Solution:** IPs changed to .11 with new DHCP server  
**Lesson:** Tracked IP changes from router to server DHCP

### **Issue 3: Understanding IP Allocation**
**Problem:** Needed clear strategy for IP assignment  
**Solution:** Created allocation plan with reserved ranges  
**Lesson:** Professional IP address planning
